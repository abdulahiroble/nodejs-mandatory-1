Hvad er HTTP?


The Hypertext Transfer Protocol (HTTP) er designet til at kunne kommunikere mellem klient og server. Det arbejder med en request/response protocol mellem klient og server. Eksempelvis sender klienten (browseren) en HTTP request til serveren, som så returnerer et response til klienten. Responset indeholder status information omkring requestet og kan også indeholde det requested indhold.


Der findes følgende HTTP metoder:


  • GET
  • POST
  • PUT
  • HEAD
  • DELETE
  • PATCH
  • OPTIONS

De mest almindelige er GET og POST.


GET


GET bruges til at requeste data fra en specifik ressource.


GET er en af de hyppigeste brugte HTTP metoder.


Forneden ser vi at vores query string (name/value par), som sendes i URL'en af et GET request:


/test/demo_form.php?name1=value1&name2=value2


POST


POST bliver brugt til at sende data til en server for at oprette/opdatere en ressource.


Dataen som sendes til serveren med POST er gemt i request bodyen af HTTP requestet:


POST /test/demo_form.php HTTP/1.1
Host: miturl.com
name1=value1&name2=value2


POST er også en af de hyppigeste brugte HTTP metoder.


PUT


PUT bruges til at sende data til en server for at oprette/opdatere en ressource.


Put kan man bruge til at oprette en ny ressource eller overskrive en allerede eksiterende en. Forskellen mellem POST og PUT er, at PUT er requests er idempotent, hvilket betyder at, hvis man kalder det samme PUT request flere gange, vil man få det samme resultat. Hvorimod kalder man det samme POST request flere gange, vil have en anden effekt, som vil oprette samme ressource flere gange.
